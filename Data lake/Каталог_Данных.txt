Добро пожаловать в Data Lake для игры “Кирпич”! Здесь происходит вся увлекательная часть действия. В нашем Data Lake есть три основных компонента:

data.parquet: В этом файле хранится вся игровая информация для нашей игры “Кирпич”. Каждая строка представляет собой событие в игре и включает следующие столбцы:

user_id: Уникальный идентификатор игрока, который сделал ход.
timestamp: Дата и время, когда был сделан ход.
player_id: Идентификатор хода игрока или компьютера
bricks_captured: Количество захваченных кирпичей в этом ходе.
bricks_remaining: Количество оставшихся кирпичей в игре после этого хода.

players.json: Этот файл содержит описание значений player_id. Благодаря нему вы модете понять сходит игрок или компьютер.

drawSQL-pgsql-export-2024-12-13: Это код SQL для создания схемы базы данных для нашей игры “Кирпич”. Если вы являетесь любителем SQL, вы можете использовать этот файл, чтобы создать собственную базу данных и анализировать данные.

А также ссылка на онлайн схему базы данных https://drawsql.app/teams/penrog/diagrams/brick-game
